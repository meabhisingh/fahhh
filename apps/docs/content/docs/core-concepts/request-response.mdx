In `fahhh`, handlers are built around standard web `Request` and `Response` objects, with a light wrapper to improve ergonomics and type safety.

## `ApiRequest`

Every exported HTTP method handler (`GET`, `POST`, etc.) receives an `ApiRequest` as its first argument.

```typescript
export type RouteParamValue = string | string[] | undefined;
export type RouteParams = Record<string, RouteParamValue>;

export interface ApiRequest<TBody = undefined, TParams = RouteParams> {
  raw: Request;
  params: TParams;
  query: URLSearchParams;
  headers: Headers;
  json(): Promise<TBody>;
}
```

This wrapper ensures that:

1. `req.json()` returns the correctly typed body.
2. `req.params` contains the typed dynamic segments of your route.
3. `req.query` exposes parsed URL search parameters via `URLSearchParams`.
4. You still have access to the raw underlying `Request` if you need it.

### Request Utilities

`fahhh` provides several utility functions to help you extract common information from the request. You can import them directly from `fahhh/utils`:

```typescript
import { headers, cookies, bearer, basicAuth } from "fahhh/utils";

export async function GET(req: ApiRequest) {
  const allHeaders = headers(req);
  const userCookies = cookies(req);
  const token = bearer(req);
  const credentials = basicAuth(req);

  // ...
}
```

## Returning Data

`fahhh` allows handlers to return raw, JSON-serializable data directly. You do not have to manually construct a `Response` object for basic endpoints.

```typescript title="src/api/hello/index.ts"
export async function GET() {
  // fahhh automatically wraps this in a JSON Response!
  return { message: "Hello World" };
}
```

### Response Escape Hatch

If you need full control over the status code, headers, or want to trigger redirects, you can return a standard `Response` object. `fahhh` provides several helpers for this:

```typescript title="src/api/auth/index.ts"
import { json, text, redirect, notFound, methodNotAllowed } from "fahhh";

export async function POST(req) {
  if (badCondition) {
    return json({ error: "Invalid input" }, { status: 400 });
  }

  if (needsLogin) {
    return redirect("/login");
  }

  if (resourceDeleted) {
    return notFound("User does not exist");
  }

  return text("Success");
}
```

Whenever a native `Response` is returned, `fahhh` will pass it through untouched to the client.

## Error Handling

Instead of manually constructing error `Response` objects, you can throw or return one of the many built-in `HttpException` classes provided by `fahhh`. The framework will automatically catch these and convert them into standard JSON error responses.

```typescript
import { NotFoundException, BadRequestException } from "fahhh/errors";

export async function GET(req) {
  if (!req.query.has("id")) {
    throw new BadRequestException("Missing ID parameter");
  }

  const user = await db.getUser(req.query.get("id"));
  if (!user) {
    throw new NotFoundException("User not found");
  }

  return user;
}
```

`fahhh` includes exceptions for all common 4xx and 5xx HTTP status codes, such as `UnauthorizedException`, `ForbiddenException`, `ConflictException`, and `InternalServerErrorException`.
